01 Unclosed tags breaking entire page layout HTML
Before
<div class="card"> <h2>Title</h2> <p>Some text <div class="footer"> ... </div>
After
<div class="card"> <h2>Title</h2> <p>Some text</p> </div> <div class="footer"> ... </div>
Fix: The <p> tag was never closed and the <div class="card"> wasn't closed either — this caused everything below to nest incorrectly inside the card, breaking the entire layout.
02 Mobile layout completely broken — content overflowing CSS
Before
.container { width: 960px; margin: 0 auto; } .card { width: 300px; float: left; }
After
.container { width: 100%; max-width: 960px; margin: 0 auto; } .card { width: 100%; max-width: 300px; }
Fix: Fixed widths in pixels don't adapt to small screens. Changed to relative units + max-width so the layout works on any screen size.
03 Button click does nothing — event listener not working JS
Before
document .getElementByid('btn') .addEventListener( 'click', sendForm() );
After
document .getElementById('btn') .addEventListener( 'click', sendForm );
Fix: Two bugs: getElementById was misspelled as "getElementByid", and sendForm() was being called immediately instead of passing the reference. Classic mistakes that are hard to spot.
